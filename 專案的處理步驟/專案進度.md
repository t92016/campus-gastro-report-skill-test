## 踩坑記錄

### 坑 12：HTTP POST 批次填表會產生幽靈資料（案件編號空白）

- **症狀**：用 PowerShell POST 模擬表單送出 35 筆，第一批（10 筆）因 checkbox 送出方式錯誤，每個症狀獨立 POST，產生 16 筆空案件編號的幽靈資料
- **原因**：多個 checkbox 值應在同一個 POST body 中以多個 `entry.XXX` 參數送出，而非分次 POST
- **解法**：修正 PowerShell 腳本，以 `&entry.XXX=值1&entry.XXX=值2` 方式一次送出所有症狀
- **後續**：管理者後台無法刪除空案件編號的資料（`adminDeleteCase` 依賴案件編號定位），需寫 `cleanupEmptyAndDupCases` 直接操作試算表

### 坑 13：Google 表單 `moveItem` 只改變 UI 順序，不改變試算表欄位落點

- **症狀**：`addHeightWeightQuestions` 執行兩次（第一次放身體症狀後，第二次放性別後），表單 UI 有重複題目
- **原因**：`form.moveItem()` 只移動表單畫面顯示順序，Google 試算表中新欄位永遠被加到最後面
- **解法**：寫 `removeDuplicateHeightWeight` 刪除表單中多餘的題目（保留最前面那題）

### 坑 14：表單連結的試算表欄位無法刪除

- **症狀**：`cleanupDuplicateSheetColumns` 嘗試 `deleteColumn` 報錯「無法刪除含有表單資料的欄」
- **原因**：Google 表單連結的試算表欄位受保護，無法透過程式刪除
- **解法**：改為隱藏欄位 + 將標題改名為 `_重複_已隱藏`，讓 `getMainColMap_` 不會認錯

### 坑 15：隱藏欄位時保留了錯誤的欄位，導致表單資料未寫入試算表（未解決）

- **症狀**：表單送出後身高體重沒出現在試算表，但管理者後台可以手動寫入
- **原因**：Google 表單永遠把新題目的資料寫到試算表「最後面」的對應欄位。`cleanupDuplicateSheetColumns` 保留了第一組身高體重欄位（舊的），把真正接收資料的最後一組身高體重欄位給隱藏了
- **嘗試解法**：寫 `restoreHeightWeightColumns` 修正（保留最後一組），但執行後仍無效
- **待查**：可能原因——`removeDuplicateHeightWeight` 刪除表單舊題目時，一併把表單與試算表欄位的連結也斷開了，新題目雖然在表單 UI 存在但試算表連結遺失。明天需確認表單題目與試算表欄位的對應關係是否需要重新建立

## 重大技術決策與理由

| 決策 | 理由 |
|------|------|
| 使用 HTTP POST 而非 Playwright 填表 | Google 表單自定義 UI 元件導致 Playwright 點擊不穩定 |
| 手機版採用卡片式佈局而非橫向捲動表格 | 與碧華國小版一致的使用者體驗 |
| 新增欄位一律透過 GAS 函式而非手動操作表單 | 確保表單與試算表聯動正確（但本次身高體重仍有遺留問題） |
| 管理者後台為唯一資料修改入口 | 避免手動動試算表造成系統欄位錯亂 |
